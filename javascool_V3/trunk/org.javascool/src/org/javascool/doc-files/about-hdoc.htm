JavaScool documentation.

Introduction.

Les documentations de JavaScool sont écrites en syntaxe XML (voir en PML si souhaité) et utilisent les structures logiques HML.
Les définitions spécifiques suivantes sont introduites.

Décrire une page HML.

Les fichiers de documentation de JavaScool ne sont pas des pages mais des textes insérés dans des pages: ils ne sont pas encapsulés par le tag page mais div.

Structurer le texte en divisions.

Les sections ({div class='..' ) de classes prédéfinies suivantes sont disposnibles:

{div class=sujet Encapsule un sujet avec, selon le besoin, les sections "objectif", "intro", "works", "notes" dans cet ordre.
{div class=objectif Section "Objectif"
{div class=intro Section "Introduction"
{div class=works Section "Travail proposé"
{div class=notes Section "Remarques"
{div class=code Qui encapsule les portions de code montrées en exemple
{div class=margin Qui des notes en marge

Enrichir le texte en modifiant son aspect.

Les constructions suivantes sont prédéfinies et permettent d'éditer des portions de code de manière colorée:

{r Mot prédéfini dans le langage ou dans la construction, il est à recopier pour utiliser l'élément.
{n Nom d'une construction logique ou d'un de ses attributs
{v Valeur textuelle d'une variable ou d'un attribut mise entre "".."
{m Ligne de commentaire qui documente le source
{T} Force une tabulation pour la mise en page du code

Créer de l'«hyper-texte» en ajoutant des objets et des liens.

Les constructions suivantes sont prédéfinies:

{l class=javascool} Entitée permettant de citer avec un lien vers le site.
{l class=proglet link=proglet-name} Insertion d'une applete de démonstration de la proglet dans la page, taille [560x720]
{l class=note link=number} Liens vers une note qui à un id numérique fixé par l'utilisateur.
