Traduction et reformulation de programmes

Traduction

PluriAlgo facilite la traduction d'un langage pédagogique (Javascool, Python, Algobox, Larp, CarMetal) vers un langage professionnel (Java, Visual Basic, Php...) ou un autre langage pédagogique :

  • recopiez le programme initial dans un éditeur (exemple Javascool, Python, Algobox, Larp, Carmetal).
  • précisez le type des variables (indication) dans l'onglet Principal, sauf si c'est déjà fait dans le programme initial (Javascool, Algobox).
  • fixez le langage du programme final (Visual Basic par exemple).
  • cliquez sur le bouton Traduire.

Le traducteur fonctionne correctement pour ce programme, ce qui ne sera évidemment pas le cas pour n'importe quel programme : donc, contrôlez le code obtenu et adaptez-le si nécessaire.

La pertinence de la traduction dépend en grande partie de l'aptitude de PluriAlgo à analyser le programme initial. C'est pourquoi vous devez respecter quelques normes de mise en forme du programme initial :

  • ne pas déclarer plusieurs variables sur une ligne.
  • ne pas écrire plusieurs instructions sur une ligne.
  • positionner les accolades comme le fait PluriAlgo quand vous utilisez les boutons Insérer.
  • ajouter des délimiteurs #end en Python (voir exemple).
  • ...

Reformulation

Ce mécanisme permet d'introduire des sous-programmes lors de la traduction :

  • recopiez le programme initial dans un éditeur (exemple Javascool, Python, Algobox, Larp, Carmetal).
  • précisez le type des variables (indication) dans l'onglet Principal, sauf si c'est déjà fait dans le programme initial (Javascool, Algobox).
  • fixez le langage du programme final (Visual Basic par exemple).
  • introduisez une fonction calculant le prix total (indication).

Diverses variantes du mécanisme de reformulation sont détaillées dans la revue MathemaTICE (voir article, section B).
