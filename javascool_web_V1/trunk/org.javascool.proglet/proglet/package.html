Ce package Java implémente les mécanismes de proglets destinés à s'initier au niveau du lycée à la programmation.

Ce package Java implémente les mécanismes de proglets JavaScool destinés à s'initier au niveau du lycée à la programmation comme détaillé ici.

Les classes utilisées par l'«élève» sont documentées en français.

Les classes utilitaires utilisées pour créer/adapater des proglets dans l'environnement Java sont documentées an anglais. Elles contiennent quelques widgets et des exemples de proglet.

Par contrat, une proglet doit:

  • Appartenir au package proglet.
  • Définir un public static final JPanel panel; de taille maximale [540 x 580] qui est l'instanciation graphique de la proglet.
  • Définir une méthode static void test() pour fournir un test/démonstration de la proglet.
  • Définir toutes les méthodes accessibles à l'utilisateur sous la forme de méthode public static.
  • Ne jamais [re]définir de méthodes accessibles à l'utilisateur de nom: clone, equals, finalize, getClass, hashCode, notify, run, toString, ou wait, réservé pour les objets Runnable de Java utilisés ici pour l'encapsulation.
Une bonne méthode pour écrire une proglet est de partir d'un exemple et de l'adapter aux nouvelles fonctionnalités souhaitées.

Ces classes sont écrites pour permettre facilement à un enseignant ou un partenaire, qui programme en Java au niveau élémentaire d'adapter, enrichir ces éléments. Non sans les partager avec nous tout(te)s en nous contactant.
