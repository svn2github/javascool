Java's Cool Guide Utilisateur : Fichier de configuration



Il y a trois fichiers de configuration situés dans un dossier JavaScoolConf (pour le trouver faites une recherche avec l'explorateur de fichier, pour windows Démarrer->Rechercher) qui comporte à l'intérieur un ensemble de fichiers dont trois qui se nomment respectivement :
- repertoire_conf.bml
- erreur_conf.bml
- fonctions_conf.bml
Pour ajouter un répertoire,une erreur ou une fonction(macro), éditez le fichier correspondant, en vous reportant à la partie concernée dans ce chapitre pour plus d'informations.


Important : Pour effectuer une modification, il suffit d’éditer le fichier de configuration, de le sauvegarder mais en aucun cas il n’est besoin de le compiler ni de l’executer. Si jamais le fichier est syntaxiquement mal écrit par exemple l'oubli d'une accolade, des messages d'erreurs apparaîtront dans la console lors du démarrage l'application.
Pensez à le sauvegarder à l'avance par crainte de fausses manipulations.
Les modifications seront prises en charge au prochain démarrage de l’application.


Répertoire

Permet de spécifier les zones de dépôt des fichiers de sauvegardes. Par exemple, si les élèves ont un dossier personnel dans "D:\Valbonne\MPI\Eleve" vous pouvez pré-enregistrer ce chemin afin que les utilisateurs n’aient pas à chaque ‘sauvegarde sous’ à naviguer depuis la racine du disque dur jusqu’à ce dossier.

L'opération d'ajout de répertoire favori peut-être directement faite via l'interface graphique avec le bouton 'Ouvrir' (voir Chapitre Interface utilisateur ). Néanmoins, si vous souhaitez le faire manuellement voici la procédure :
Dans le fichier "repertoire_conf.bml" dans la partie ’répertoires’, ajoutez un identifiant sous la forme 'idX{}' où X est un nombre non utilisé par les précédents identifiants. Puis entre les deux accolades donnez un nom à la configuration sous la forme 'name="..."'. C’est celui qui s’affichera dans l’interface utilisateur, il est donc important de choisir un nom explicite. Une fois choisi, en dessous mettez le mot "path=" et indiquez le chemin absolu entre guillemets.

Exemple : Si mon chemin absolu est "D:\Valbonne\MPI\Eleve" et que je souhaite rajouter ce répertoire, cela donne :


Outils {
Répertoires {
…
idX{
nom="élève";
path="D:\Valbonne\MPI\Eleve";
}
…
}
}



Configurer les erreurs

Afin d’avoir une traduction et une compréhension des erreurs qui soit simple pour les élèves, il est laissé aux encadrants la possibilité d’ajouter des messages pour les erreurs qui pourraient survenir lors de la compilation ou de l’exécution du programme. Pour cela, il faut ajouter l’erreur ainsi que sa traduction dans la partie ‘Erreurs’. Si l’erreur d’exécution n’a pas été ajoutée et qu’elle survient, le message d’erreur sera en anglais dans la console.

Manipulation pour ajouter un message d'erreur : Dans le fichier "erreur_conf.bml" dans la partie 'Erreurs', ajoutez un identifiant sous la forme 'idX{}' où X est un nombre non utilisé par les précédents identifiants. Puis entre les deux accolades donnez le nom de l'erreur java sous la forme 'java="..."'. C’est celui qui s’affichera dans l’interface utilisateur, il est donc important de choisir un nom explicite. Une fois inscrit, mettez la traduction ('traduction=') que vous souhaitez attribuer au message d'erreur.


Exemple : Si mon erreur est "IndexOutOfBoundsException" et que je souhaite rajouter cette erreur, cela donne :


Outils {
Erreurs {
…
idX{
java="IndexOutOfBoundsException";
traduction="Vous essayez d'atteindre une case de tableau qui n'existe pas.";
}
…
}
}

Où puis-je trouver de nouveaux messages d'erreur à traduire ? :
Deux solutions, attendez qu'elles se présentent à vous ou allez sur la documentation officielle Sun pour JAVA 1.6 et arpentez les divers packages des fonctions que vous avez dans la partie 'Fonctions' via la fenêtre en haut à gauche. Dans la fenêtre en dessous faites défiler le nom des classes pour arriver à la rubrique 'errors', vous aurez alors la descriptions des erreurs que peut renvoyer la classe.

Comment trouver la signification d'une erreur ? :
Rendez-vous sur la documentation officielle Sun pour JAVA 1.6. Dans la fenêtre principale cliquez sur index. Il vous est proposé par ordre alphabétique tous les noms apparaissant dans la documentation dont les classes d'erreurs. Une fois trouvé, cliquez dessus pour obtenir la cause qui peut déclencher l'erreur. Avec de l'expérience vous cernerez vite la raison d'une erreur.


Configurer les macros

Permet de rajouter de nouvelles fonctions dans la boîte à Outils. Les fonctions définies dans cette partie sont les macros qui seront traduites lors de la phase de compilation.

Manipulation pour ajouter une maccro : Dans le fichier "fonctions_conf.bml" dans la partie 'Fonctions', ajoutez un identifiant sous la forme d'un nom de fonction unique, c'est celui qui apparaîtra dans la boîte à outils. Ouvrez les accolades et renseignez tous les champs à savoir :

'desc=' qui est la description qui s'affiche quand le curseur reste sur la fonction.

'import=' le package java d'où provient la fonction.

'signature=' la signature de la fonction qui est composée du type de retour, du nom original de la fonction et de ses paramètres (exemple : double max(double a,double b) ici le type de retour est un double, le nom de la fonction est max et elle prend deux doubles en paramètres. La fonction renvoyant le plus grand des deux dans son type retour).

'type=' la catégorie à laquelle appartient la fonction. C'est celle qui s'affichera dans la boîte à outils. Vous pouvez en créer autant qu'il vous plaira.


Exemple : Si ma fonction est dans la classe "math" du package "java.lang" et si je souhaite la rajouter, cela donne :


Outils {
Fonctions {
…
maFonction{
desc="ma fonction";
import="java.math";
signature="void maFonction(int a)";
type="math";
}
…
}
}


Où puis-je trouver de nouvelles fonctions ? :

Rendez-vous dans la documentation officielle Sun pour JAVA 1.6.
La fenêtre du navigateur se sépare en trois frames, la première située dans le coin haut-gauche permet de naviguer via l'ascenseur au sein de tous les packages java. Les packages sont pour simplifier, un ensemble de classes réunies dans un même groupe. Dans la frame, le long du bord gauche se trouve l'ensemble des classes contenues. Pour chacune d'entre elles vous pouvez avoir une description au sein de la page centrale, avec en haut des informations générales et plus bas dans la page, l'ensemble des fonctions disponibles.

Les informations qui vous intéressent sont l'import à effectuer qui correspond au "nom du package'.'le nom de la classe" (exemple la classe math se situe dans le package java.lang, l'import à effectuer est : java.lang.math). Il peut aussi être trouvé en haut de la page centrale quand on visualise une classe (java.lang.Object |-> java.lang.Math).
Il vous faut aussi le nom de la fonction/method avec son type de retour (exemple : double abs(double a)). Vous n'avez pas besoin de mettre les mots static ni public avant le type de retour par contre une méthode dite private et protected ne peut pas être mise en Macro car elle n'est pas censée être utilisée en dehors de son implémentation au sein de la classe où elle a été crée.
